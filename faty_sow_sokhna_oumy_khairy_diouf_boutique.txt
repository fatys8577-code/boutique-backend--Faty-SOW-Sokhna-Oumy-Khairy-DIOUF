# Présentation des différentes vues de l'application (Frontend)

Le frontend de notre application a été développé avec **React**, **Vite**, **React Router** et **Tailwind CSS**. Il communique avec l'API REST développée en Laravel afin de récupérer et d'afficher les données de la boutique.

L'interface a été conçue pour être simple, moderne, responsive et facile à utiliser aussi bien sur ordinateur que sur mobile.

Les principales vues de l'application sont les suivantes :

## 1. Page d'accueil

Cette page constitue le point d'entrée de l'application. Elle présente le projet de gestion de boutique et permet d'accéder rapidement aux principales fonctionnalités grâce à une barre de navigation commune.

## 2. Liste des produits

Cette page affiche tous les produits enregistrés dans la base de données. Pour chaque produit, l'utilisateur peut consulter :

* le nom ;
* le prix ;
* le stock disponible ;
* la catégorie associée.

Les produits sont présentés sous forme de cartes modernes avec une mise en page responsive.

## 3. Détail d'un produit

Cette vue présente toutes les informations concernant un produit sélectionné :

* son nom ;
* son prix ;
* son stock ;
* sa catégorie ;
* sa description lorsqu'elle est disponible.

Un bouton permet également de revenir facilement à la liste des produits.

## 4. Liste des catégories

Cette page affiche toutes les catégories enregistrées dans la boutique.

Chaque catégorie présente :

* son nom ;
* sa description ;
* le nombre de produits appartenant à cette catégorie.

Les catégories sont également affichées sous forme de cartes afin d'assurer une présentation claire et homogène.

## Interface utilisateur

L'ensemble de l'interface a été réalisé avec **Tailwind CSS**.

Elle comprend notamment :

* une barre de navigation commune à toutes les pages ;
* une page d'accueil moderne ;
* des cartes pour les produits et les catégories ;
* des couleurs harmonieuses ;
* des effets visuels au survol des cartes ;
* un pied de page (Footer) affichant le nom du projet ainsi que les membres du binôme ;
* une interface responsive adaptée aux ordinateurs et aux appareils mobiles.

# Présentation de la documentation Swagger (Backend)

Le backend a été développé avec **Laravel** et expose une **API REST** documentée grâce à **Swagger**.

Cette documentation interactive permet de consulter et de tester directement toutes les routes de l'API sans utiliser un logiciel externe.

Les principales ressources documentées sont :

## Catégories

* Lister les catégories
* Créer une catégorie
* Consulter une catégorie
* Modifier une catégorie
* Supprimer une catégorie

## Produits

* Lister les produits
* Créer un produit
* Consulter un produit
* Modifier un produit
* Supprimer un produit

## Acheteurs

* Lister les acheteurs
* Créer un acheteur
* Consulter un acheteur
* Modifier un acheteur
* Supprimer un acheteur
* Enregistrer un achat

Swagger permet également de visualiser :

* les paramètres des requêtes ;
* les réponses de l'API au format JSON ;
* les différents codes de réponse HTTP.

# Les différents rôles de l'application

L'application met en œuvre une gestion des rôles afin de limiter l'accès aux différentes fonctionnalités.

## Administrateur

L'administrateur possède tous les droits. Il peut :

* consulter les catégories, les produits et les acheteurs ;
* créer, modifier et supprimer les catégories ;
* créer, modifier et supprimer les produits ;
* créer, modifier et supprimer les acheteurs ;
* gérer les utilisateurs et leurs rôles.

## Gestionnaire

Le gestionnaire peut :

* consulter les catégories, les produits et les acheteurs ;
* créer, modifier et supprimer les catégories ;
* créer, modifier et supprimer les produits ;
* créer, modifier et supprimer les acheteurs ;
* enregistrer les achats.

## Employé

L'employé peut :

* consulter les catégories ;
* consulter les produits ;
* consulter les acheteurs ;
* enregistrer les achats.

Les restrictions d'accès sont appliquées aussi bien sur les routes Laravel que sur l'affichage des différentes vues de l'application.
