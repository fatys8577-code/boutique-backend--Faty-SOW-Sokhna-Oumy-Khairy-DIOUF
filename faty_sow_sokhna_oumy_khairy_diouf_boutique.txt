## Présentation des différentes vues de l'application (Frontend)

Le frontend de notre application a été développé avec **React**, **Vite**, **React Router** et **Tailwind CSS**. Il permet aux utilisateurs de consulter les informations de la boutique à travers une interface simple et moderne.

Les principales vues de l'application sont les suivantes :

### 1. Page d'accueil

Cette page constitue le point d'entrée de l'application. Elle présente l'application de gestion de boutique et permet d'accéder facilement aux différentes fonctionnalités grâce à la barre de navigation.

### 2. Liste des produits

Cette page affiche tous les produits enregistrés dans la base de données. Pour chaque produit, l'utilisateur peut consulter son nom, son prix, son stock disponible ainsi que sa catégorie.

### 3. Détail d'un produit

Cette vue présente toutes les informations concernant un produit sélectionné. Elle permet de consulter ses caractéristiques ainsi que la catégorie à laquelle il appartient.

### 4. Liste des catégories

Cette page affiche toutes les catégories disponibles dans la boutique. Elle facilite la consultation et l'organisation des différents produits.

La navigation entre ces différentes pages est assurée grâce à React Router afin d'offrir une expérience fluide à l'utilisateur.

---

## Présentation de la documentation Swagger (Backend)

Le backend a été développé avec **Laravel** et expose une **API REST** entièrement documentée grâce à **Swagger**.

Cette documentation interactive permet de consulter et de tester directement toutes les routes de l'API sans utiliser un outil externe.

Les principales ressources documentées sont :

### Catégories

* Lister les catégories
* Créer une catégorie
* Consulter une catégorie
* Modifier une catégorie
* Supprimer une catégorie

### Produits

* Lister les produits
* Créer un produit
* Consulter un produit
* Modifier un produit
* Supprimer un produit

### Acheteurs

* Lister les acheteurs
* Créer un acheteur
* Consulter les informations d'un acheteur
* Modifier un acheteur
* Supprimer un acheteur
* Enregistrer un nouvel achat pour un acheteur

Swagger permet également de visualiser les paramètres attendus, les réponses de l'API ainsi que les différents codes HTTP retournés.

## Les différents rôles de l'application

L'application met en œuvre une gestion des rôles afin de contrôler les accès aux différentes fonctionnalités.

### Administrateur

L'administrateur possède tous les droits. Il peut :

* consulter les catégories, produits et acheteurs ;
* créer, modifier et supprimer les catégories ;
* créer, modifier et supprimer les produits ;
* créer, modifier et supprimer les acheteurs ;
* gérer les utilisateurs ainsi que leurs rôles.

### Gestionnaire

Le gestionnaire peut :

* consulter les catégories, produits et acheteurs ;
* créer, modifier et supprimer les catégories ;
* créer, modifier et supprimer les produits ;
* créer, modifier et supprimer les acheteurs ;
* enregistrer les achats.

### Employé

L'employé peut :

* consulter les catégories ;
* consulter les produits ;
* consulter les acheteurs ;
* enregistrer un achat.

Les restrictions sont appliquées aussi bien au niveau des routes Laravel que de l'affichage des différentes vues.

 Conclusion

Ce projet nous a permis de mettre en pratique les principaux concepts de Laravel et de React, notamment la création d'une API REST, la gestion des relations entre les modèles, l'authentification, les rôles utilisateurs, la documentation avec Swagger ainsi que le développement d'une interface moderne consommant les données de l'API.

L'ensemble de l'application respecte les exigences fonctionnelles définies dans la consigne et permet la gestion des catégories, des produits, des acheteurs ainsi que des achats.
